# Pro 教程怎么学

这一页主要帮你解决一个问题：**如果你第一次用 DVHA Pro，应该按什么顺序学习。**

如果你不想东一页西一页地跳着看，建议直接按下面顺序来。

## 推荐学习顺序

### 1. 先把项目跑起来

先看：[`快速开始`](/pro/course/quick-start)

这一页适合你：

- 想先把 Pro 项目跑起来
- 想知道最短上手路径
- 想先看到一个能跑的后台项目

如果你现在还没跑起来，建议先不要往后看，先把这一步完成。

### 2. 再做第一个页面

再看：[`第一个页面`](/pro/course/start)

这一页适合你：

- 已经把项目跑起来了
- 想知道页面是怎么接进去的
- 想真正开始写自己的业务页面

这一步完成后，你就会对“Pro 项目里页面是怎么组织的”有基本感觉。

### 3. 然后补登录和鉴权

再看：[`登录与鉴权`](/pro/course/login)

这一页适合你：

- 想把登录流程接起来
- 想知道登录态和未登录态怎么切换
- 想知道后台项目最基础的鉴权怎么处理

后台项目几乎都会走到这一步，所以建议尽早看。

### 4. 接你自己的数据接口

再看：[`自定义数据接口`](/pro/course/api)

这一页适合你：

- 想把假数据替换成真实接口
- 想知道数据请求怎么和业务 API 对上
- 想开始做真正的列表、表单、详情页面

这是“从会跑”到“能做业务”的关键一步。

### 5. 再做权限和路由控制

再看：[`权限管理与路由守卫`](/pro/course/permission)

这一页适合你：

- 项目开始有角色区分了
- 不同用户要看不同页面了
- 菜单和路由需要按权限控制了

如果你的项目是中后台，这一步通常迟早要做。

### 6. 再补常见业务能力

接下来可以按业务需要继续看：

- [`文件上传与管理`](/pro/course/upload)
- [`JSON Schema 动态表单`](/pro/course/json-schema)
- [`远程组件与微前端`](/pro/course/remote)

可以这样理解：

- **上传**：适合做媒体、附件、文件类业务
- **JSON Schema**：适合做动态表单、配置化页面
- **远程组件**：适合做动态页面、微前端、远程扩展

## 如果你时间不多，最短学习路径是什么

如果你只想先快速上手，建议只看这四页：

1. [`快速开始`](/pro/course/quick-start)
2. [`第一个页面`](/pro/course/start)
3. [`登录与鉴权`](/pro/course/login)
4. [`自定义数据接口`](/pro/course/api)

这四页看完，通常就已经足够开始做一个基础后台项目。

## 如果你要做更复杂的后台

如果你的项目还有下面这些需求：

- 多角色权限
- 文件上传
- 动态表单
- 远程页面
- 微前端扩展

那建议把后面的教程也按顺序继续看完。

## 学完教程后再看什么

教程看完之后，通常可以继续看：

- Pro 页面组件：[`/pro/pages`](/pro/pages)
- Pro 组件库：[`/pro/components/`](/pro/components/)
- Pro Hooks：[`/pro/hooks/`](/pro/hooks/)
- Pro 配置说明：[`/pro/configuration`](/pro/configuration)
- Core 和 Pro 怎么选：[`/pro/choose`](/pro/choose)

## 最后一句建议

如果你是第一次接触 DVHA Pro，最好的方式不是把所有文档一次看完，

而是：**先跑起来，再做一个页面，再接一个接口。**

这样理解会快很多。
