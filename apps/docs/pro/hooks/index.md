# Pro Hooks 先看什么

这一页主要讲：**DVHA Pro 的 Hooks 能帮你少写哪些交互代码。**

如果组件解决的是“页面怎么搭”，那 Hooks 解决的通常就是：

- 交互怎么管
- 状态怎么收
- 弹窗怎么开
- 抽屉怎么控
- 表格怎么增强

## 如果你在做表格页

优先看：

- [`表格 Hooks`](/pro/hooks/table)
- [`操作增强`](/pro/hooks/action)

适合你：

- 列表页操作很多
- 想把表格交互做得更顺
- 想把按钮、批量操作、状态切换收敛起来

## 如果你在做弹窗和抽屉

优先看：

- [`对话框管理`](/pro/hooks/dialog)
- [`抽屉管理`](/pro/hooks/drawer)
- [`模态框管理`](/pro/hooks/modal)

适合你：

- 想统一管理弹窗开关
- 想做编辑抽屉、详情弹窗
- 想减少手写状态变量和重复逻辑

## 如果你在做下载、图片、图表

优先看：

- [`下载功能`](/pro/hooks/download)
- [`图片处理`](/pro/hooks/image)
- [`图表处理`](/pro/hooks/echart)

适合你：

- 导出文件
- 图片预览 / 处理
- 图表配置和渲染辅助

## 如果你在做层级或 UI 工具能力

优先看：

- [`层级处理`](/pro/hooks/level)
- [`UI 工具`](/pro/hooks/ui)

适合你：

- 树级结构处理
- 一些偏页面交互层的辅助逻辑

## 如果你不确定先看什么

最常用、最值得先看的通常是：

1. [`表格 Hooks`](/pro/hooks/table)
2. [`操作增强`](/pro/hooks/action)
3. [`对话框管理`](/pro/hooks/dialog)
4. [`抽屉管理`](/pro/hooks/drawer)

如果你平时主要做后台 CRUD，这几组 Hooks 使用频率通常最高。

## 最后一句理解

Pro Hooks 的价值，不只是“封装几个函数”，

而是帮你把后台里最容易重复的交互管理逻辑尽量收起来。
